# Test Screenshots Directory

This directory contains screenshots and test reports generated by the responsive testing suite.

## Directory Contents

When you run the responsive tests, this directory will contain:

### Screenshots (PNG files)
- `chrome_mobile_small_YYYYMMDD_HHMMSS.png`
- `chrome_mobile_medium_YYYYMMDD_HHMMSS.png`
- `chrome_tablet_portrait_YYYYMMDD_HHMMSS.png`
- `chrome_desktop_medium_YYYYMMDD_HHMMSS.png`
- etc.

**Naming Convention**: `{browser}_{screen_size}_{timestamp}.png`

### Test Reports (JSON files)
- `responsive_test_report_YYYYMMDD_HHMMSS.json`

**Format**: Structured JSON with test results, metrics, and summary

## File Types

### Screenshot Files
- **Format**: PNG
- **Purpose**: Visual documentation of test results
- **Usage**: Manual review, regression testing, documentation

### Report Files
- **Format**: JSON
- **Purpose**: Structured test results and metrics
- **Usage**: Automated analysis, CI/CD integration, tracking

## Viewing Results

### View Screenshots
```bash
# List all screenshots
ls -lt test_screenshots/*.png

# Open latest screenshot (Linux)
xdg-open $(ls -t test_screenshots/*.png | head -1)

# Open latest screenshot (macOS)
open $(ls -t test_screenshots/*.png | head -1)
```

### View Reports
```bash
# View latest report with jq
cat $(ls -t test_screenshots/*.json | head -1) | jq .

# View summary only
cat $(ls -t test_screenshots/*.json | head -1) | jq .summary

# View test results
cat $(ls -t test_screenshots/*.json | head -1) | jq .results
```

## Cleanup

### Remove Old Files
```bash
# Remove files older than 7 days
find test_screenshots/ -type f -mtime +7 -delete

# Remove all test files
rm test_screenshots/*.png test_screenshots/*.json
```

### Keep Latest Only
```bash
# Keep only last 5 screenshots per browser
cd test_screenshots
for browser in chrome firefox edge safari; do
  ls -t ${browser}_*.png 2>/dev/null | tail -n +6 | xargs -r rm
done
```

## File Size Management

- Screenshots: ~50-500 KB each
- Reports: ~10-50 KB each
- Recommended: Keep last 10 runs (~5-10 MB)

## Ignore in Git

This directory is included in `.gitignore` to avoid committing large binary files:

```gitignore
# In .gitignore
test_screenshots/*.png
test_screenshots/*.json
```

## CI/CD Integration

### Store as Artifacts
```yaml
# Example: GitHub Actions
- name: Upload test screenshots
  uses: actions/upload-artifact@v3
  with:
    name: responsive-test-results
    path: test_screenshots/
    retention-days: 30
```

### Archive Results
```bash
# Create archive of latest test run
timestamp=$(date +%Y%m%d_%H%M%S)
tar -czf test_results_${timestamp}.tar.gz test_screenshots/
```

## Troubleshooting

### No Screenshots Generated
- Ensure browser is running correctly
- Check write permissions on directory
- Verify Selenium driver is installed
- Run with `headless=False` to debug

### Large File Sizes
- Screenshots are full-page captures
- Reduce window size to reduce file size
- Use compression tools if needed
- Clean up old files regularly
